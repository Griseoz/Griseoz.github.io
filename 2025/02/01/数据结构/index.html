数据结构

数组

核心思想：一段连续的内存空间，用于存储相同类型的多个元素。

特点：

  • 随机访问：通过下标（索引）可以直接访问任何一个元素，时间复杂度为 O(1)这是数组最大的优势。
  • 固定大小：在创建时通常需要指定大小，后续不易动态扩展（但在一些高级语言- 中，有动态数组的实现，如 C++ 的 vector，Java 的 ArrayList）。
  • 内存紧凑：因为是连续存储，所以内存利用率高，CPU 缓存友好。

缺点：

  • 在数组中间进行插入或删除操作时，需要移动后续的所有元素，平均时间复杂度为 O(n)

字符串

核心思想：在大多数编程语言中，字符串本质上就是一个字符数组。

特点：

  • 具有数组的所有特性（连续存储、随机访问）。
  • 通常以特殊的终止符（如 C/C++ 中的 ‘\0’）标识结尾，或者附带一个长度字段（如 Java、Python）。
  • 针对文本操作有许多专门的算法，如字符串匹配（KMP 算法）、拼接、分割、查找等。

注意：由于其特殊性，字符串常常被单独作为一种数据类型来研究和处理。

链表

核心思想：通过指针（或引用）将一组零散的内存块串联起来。每个节点包含 数据 和 指向下一个节点的指针。

种类：

  • 单向链表：节点只有一个指针 next，指向下一个节点。
  • 双向链表：节点有 prev 和 next 两个指针，分别指向前一个和后一个节点。
  • 循环链表：尾节点的 next 指向头节点，形成一个环。

特点：

  • 动态大小：可以非常方便地动态添加和删除节点，无需预先分配固定空间。
  • 高效插入/删除：在知道某个节点的位置后，进行插入或删除操作的时间复杂度为 O(1)。
  • 非连续存储：内存不要求连续，可以充分利用零散内存。

缺点：

  • 随机访问低效：要访问第 i 个元素，必须从头节点开始一个一个遍历，时间复杂度为 O(n)。
  • 需要额外的空间来存储指针

栈

核心思想：一种操作受限的线性表，只允许在一端（称为栈顶）进行插入（入栈/Push）和删除（出栈/Pop）操作。

特点：

  • 后进先出：最后一个放入的元素，第一个被取出来。
  • 操作非常简单，只有 push, pop, peek（查看栈顶元素但不弹出）。

应用场景：

  • 函数调用栈（记录函数调用时的临时变量、返回地址等）。
  • 表达式求值（如检查括号是否匹配）。
  • 浏览器的前进/后退功能。

实现：可以用数组或链表来实现

队列

核心思想：另一种操作受限的线性表，只允许在一端进行插入（入队/Enqueue），在另一端进行删除（出队/Dequeue）。

特点：

  • 先进先出：就像排队，先来的人先接受服务。

种类：

  • 普通队列。
  • 双端队列：两端都可以进行入队和出队操作。
  • 循环队列：将数组的首尾相连，解决“假溢出”问题，更高效地利用空间。
  • 优先队列：出队顺序不是由入队时间决定，而是由元素的“优先级”决定，通常用堆来实现。

应用场景：

  • 消息队列。
  • 线程池的任务等待队列。
  • 广度优先搜索（BFS）算法。

实现：也可以用数组或链表来实现

集合

核心思想：存储不重复元素的容器。它更侧重于元素的“存在性”检查。

核心操作：

  • add(element)：添加元素，如果已存在则不再添加。
  • delete(element)：删除元素。
  • has(element)：检查元素是否存在（非常关键的操作）。

特点：

  • 元素唯一性。
  • 不强调顺序：通常不关心元素存入的顺序（但有些实现如 LinkedHashSet 会维护顺序）。
  • 实现：通常基于哈希表（效率最高，O(1) 时间进行存在性判断）或树（如红黑树，元素会自动排序，操作时间复杂度为 O(log n)）来实现

哈希表

核心思想：通过一个哈希函数将键映射到一个数组中的某个位置，从而可以直接通过键来访问值。

核心组成部分：

  • 哈希函数：将任意大小的输入（键）转换为一个固定范围的数组索引。好的哈希函数应尽量均匀分布，减少冲突。

  • 数组：存储数据的桶。

关键问题：

哈希冲突：不同的键可能被映射到同一个数组索引上。

解决方法：

  • 链地址法：每个数组位置不再存放单个元素，而是存放一个链表（或其他数据结构）。发生冲突的元素都被放到这个链表中。

  • 开放寻址法：当发生冲突时，按照某种规则（如线性探测、二次探测）寻找数组中下一个空位。

特点：

  • 在理想情况下，插入、删除、查找的平均时间复杂度都是 O(1)，性能极高。
  • 是无序的。

应用场景：

  • 实现字典或映射（如 Python 的 dict，Java 的 HashMap）。
  • 实现集合（如 Python 的 set，Java 的 HashSet）。
  • 缓存（如 Redis）

二叉树

核心思想：一种非线性的树状结构。每个节点最多有两个子节点，称为左子节点和右子节点。

相关术语：

  • 根节点：最顶层的节点。
  • 叶节点：没有子节点的节点。
  • 深度：从根节点到该节点所经历的边的个数。
  • 高度：从该节点到最远叶节点的最长路径上的边数。

重要种类：

二叉搜索树：

  • 定义：对于任意节点，其左子树上所有节点的值都小于该节点的值；其右子树上所有节点的值都大于该节点的值。

  • 作用：可以非常高效地进行查找、插入、删除操作，平均时间复杂度为 O(log n)。（如果树退化成一条链表，最坏情况为 O(n)）

平衡二叉搜索树：

  • 定义：通过特定的旋转操作（如 AVL 树、红黑树），使二叉搜索树始终保持平衡（左右子树高度差不超过 1）。

  • 作用：解决了 BST 可能退化成链表的问题，将最坏情况下的操作时间复杂度也稳定在 O(log n)。

遍历方式（非常重要）：

  • 前序遍历：根 -> 左 -> 右
  • 中序遍历：左 -> 根 -> 右 （对 BST 进行中序遍历，可以得到一个升序序列）
  • 后序遍历：左 -> 右 -> 根
  • 层序遍历：按层次从上到下、从左到右遍历（通常用队列实现）。

应用场景：

  • 文件系统的目录结构。
  • 数据库的索引。
  • 高效的数据查找和排序
