接口和自动化接口测试

1. 接口

接口是由后端开发人员编写的系统功能入口，调用接口实现 service 服务。在软件工程中，接口是两个独立系统或组件之间进行交互和通信的桥梁。它严格定义了：

  • 如何请求服务（比如：“调用哪个函数？”、“访问哪个网址？”）
  • 需要提供什么数据（比如：“参数叫什么名字？是什么格式？”）
  • 会返回什么结果（比如：“返回的是 JSON 数据吗？里面包含哪些字段？”）

举个例子，你想要给手机充电会怎么做。一般来说是拿着充电器找一个插座，手机连接充电器，充电器插在插座上，所有设备正常且适配的情况就充上电了。在这里手机与充电器之间，充电器与插座之间就存在接口。这个“插座” 就是一个接口。它定义了一套规范（比如电压、电流、插孔形状），只要你的设备符合这个规范，就能获得电力服务。

3. 自动化接口测试

什么是自动化接口测试

自动化接口测试 (Automated API Testing) 就是通过编写代码和脚本，模拟客户端向服务器发送请求，并自动验证返回的响应结果是否符合预期的一种测试方法。这里说的是使用 python 编码测试脚本，属于测试开发，当然使用接口测试工具也是属于接口自动化的。简单来说自动化就是使用测试工具或者脚本填入接口文档的四要素 (url + method + headers + body)即可访问。

一个简单的自动化测试脚本可能做的事：

  1. 构造请求：模拟一个用户登录的请求，用户名是 _testuser_，密码是 123456。

  2. 发送请求：发送到服务器的登录接口 POST /api/login。

  3. 验证响应：

  • 断言 (Assert) 状态码是否是 200（成功）。
  • 断言 返回的 JSON 数据中是否包含 “success”: true。
  • 断言 是否返回了一个有效的用户 _token_。
  1. 输出结果：如果所有断言都通过，标记测试用例为“成功”；如果有任何一项失败，则标记为“失败”并打印出详细错误信息。

为什么做自动化接口测试

自动化的特点

  • 高效率：可以在几分钟内运行成百上千个测试用例。

  • 高可靠性：避免人为疏忽和错误，检查更全面精准。

  • 可持续性：非常适合集成到 CI/CD 流程中，每次代码提交都能自动测试，快速发现回归缺陷。

  • 低成本：虽然前期编写脚本有投入，但长期来看，大大减少了重复的手工测试工作。

从需求来说，现在互联网企业微服务复杂，需要大量的接口测试，人工完成一些重复的接口测试用例不如自动化性价比高。测试人员加强Python 编程以适应企业方式。

如何掌握自动化接口测试

作为一个测试开发需要对 python 的库使用熟悉，以下是一些必须掌握的库：

  • Requests: Python 发出接口访问请求的
  • Pytest: 数据驱动 组织测试用例
  • logging: 日志库
  • Pymysql: Python 读取 mysql 中的数据的（Excel 编写测试用例转变到mysql 测试用例 ）

加强 Python 语法，学习企业编程方式 —— API 表 和 工具类封装 （代码不是一行行 ，主要会复用）

综合应用，写一套自定义框架加载用例执行目标： 无人值守（ git+jenkins ） 产出报告（ allure ）
