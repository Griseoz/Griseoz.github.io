Spring的依赖注入（DI）和控制反转（Ioc）
发表于2024-02-10|Java|Java•DI•Ioc
Spring 的依赖注入（DI）和控制反转（Ioc）DI-依赖注入 创建对象实例时，同时为这个对象注入它所依赖的属性。相当于把每个bean与 bean 之间的关系交给容器管理。而这个容器就是 spring。好这里说到 bean，那我们来说明下什么是 bean bean 在 Spring 中，构成应用程序主干并由 Spring IoC 容器管理的对象称为 bean。bean 是由 Spring IoC 容器实例化、组装和管理的对象。本质上指代任何被 Spring 加载生成出来的对象。 Ioc-控制反转 控制反转通过依赖注入（DI）方式实现对象之间的松耦合关系。程序运行时，依赖对象由辅助程序动态生成并注入到被依赖对象中，动态绑定两者的使用关系。Spring IoC 容器就是这样的辅助程序，它负责对象的生成和依赖的注入，然后再交由我们使用。

java 反射反射的概念 指在运行时动态地获取类的信息，并可以通过该信息来操作类或对象。这是 java 的一种机制，通过反射机制可以在运行时获取取类的字段、方法、构造函数等信息，并能够动态地创建对象、调用方法、访问和修改字段的值。 反射的原理 基于 Java 的运行时数据区域（Runtime Data Area）和类加载机制。当 JVM 加载一个类时，它将类的字节码文件加载到内存中，并在方法区创建一个 Class 对象来表示该类。Class 对象包含了类的完整信息，包括类的构造函数、方法、字段等。
