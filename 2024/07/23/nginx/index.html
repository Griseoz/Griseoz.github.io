什么是 nginx

Nginx 是一款轻量级的高性能 HTTP 和反向代理 web 服务器，也可以作为电子邮件（IMAP/POP3）代理服务器和通用的 TCP/UDP 代理服务器。

nginx 的特点

Nginx 的成功源于其独特的设计和诸多优点，主要优点是高性能、高并发、低内存消耗和稳定性：

  1. 高性能、高并发：
  • 事件驱动、异步非阻塞架构：这是 Nginx 的核心优势。它使用一个（或少数几个）工作进程（worker process）来处理成千上万的并发连接，而不是为每个连接创建一个新的线程或进程。这极大地降低了 CPU 和内存的消耗。
  • 内存占用极低：处理静态资源时，每个连接仅占用极少量内存。
  1. 高可靠性 & 稳定性：
  • 经过大规模网站的实战检验，可以长时间稳定运行。
  • 具备良好的容错机制，后端服务器故障时能自动剔除。
  1. 丰富的功能集：
  • HTTP 服务器：高效处理静态文件（如 HTML, CSS, JS, 图片）。
  • 反向代理与负载均衡：将客户端请求转发给后端的多台应用服务器（如 PHP, Python, Java 应用），并分配负载。
  • SSL/TLS 终止：处理 HTTPS 加密和解密，减轻后端服务器的压力。
  • 缓存：可以缓存反向代理的内容，加速访问。
  • 压缩（Gzip）：压缩响应内容，减少网络传输时间。
  • 虚拟主机：基于域名或 IP 在一台服务器上托管多个网站。
  1. 高度模块化 & 可扩展：
  • 核心功能通过模块实现，用户可以根据需要选择编译或动态加载模块。
  • 有丰富的第三方模块生态。

nginx 能做什么

  1. 作为静态内容服务器（Web Server）
  • 这是它的老本行。处理 HTML、CSS、JavaScript、图片等静态文件的速度极快，效率远高于传统应用服务器。通常架构是：动态请求交给应用服务器处理，静态请求直接由 Nginx 返回。
  1. 作为反向代理服务器（Reverse Proxy）
  • 这是目前最主流的用法。Nginx 接收客户端的请求，然后根据配置规则，将请求转发到内部网络的一台或多台应用服务器（如 Tomcat, Django, Node.js, PHP-FPM 等），并将应用服务器的响应返回给客户端。
  • 好处：
    • 隐藏后端服务：增强安全性。
    • 负载均衡：将流量均匀分配到多台后端服务器，提高系统处理能力和可靠性。
    • 易于扩展：可以方便地增减后端服务器。
  1. 作为负载均衡器（Load Balancer）

  • 这是反向代理功能的延伸。Nginx 支持多种负载均衡策略：
    • 轮询 (Round Robin)：默认方式，按顺序分配请求。
    • 权重 (Weight)：给性能好的服务器分配更高权重的流量。
    • IP 哈希 (IP Hash)：根据客户端 IP 计算哈希值，将同一 IP 的请求总是发给同一台后端服务器（用于解决 Session 保持问题）。
    • 最少连接 (Least Connections)：将请求发给当前连接数最少的后端服务器。
  1. 提供 API 网关功能
  • 在现代微服务架构中，Nginx 常被用作 API 网关，统一管理、路由、认证和限流所有进入微服务集群的请求。
  1. 处理 SSL 终端
  • 所有 HTTPS 的加密解密工作都在 Nginx 这一步完成，后端服务器只需处理明文的 HTTP 请求，简化了后端应用的复杂性。
