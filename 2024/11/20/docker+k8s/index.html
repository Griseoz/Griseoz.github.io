docker

Docker 是什么？

Docker 是一个开源的应用容器引擎。它允许开发者将应用以及应用的所有依赖（库、环境配置、系统工具等）打包到一个轻量级、可移植的容器中。这个容器可以在任何安装了 Docker 的机器上运行，从而保证了环境的一致性。

核心概念

镜像 (Image)：

一个只读的模板，类似于面向对象中的“类”。它包含了运行应用所需的文件系统结构和内容（代码、运行时、库、环境变量、配置文件等）。

例如：一个 Ubuntu 镜像、一个包含 Java 和 Tomcat 的镜像、一个包含你的 SpringBoot 应用的镜像。

镜像是通过 Dockerfile 文件构建出来的。

容器 (Container)：

镜像的一个运行实例，类似于一个“对象”。容器是独立、安全、隔离的进程。

你可以创建、启动、停止、删除容器。每个容器之间相互隔离。

基于同一个镜像可以运行多个互不干扰的容器。

仓库 (Registry)：

用来存储和分发 Docker 镜像的地方，最著名的是 Docker Hub（类似于 GitHub for code）。

你可以从仓库拉取（pull）公共镜像（如 nginx, redis），也可以将你自己构建的镜像推送（push）到仓库供他人使用。

Dockerfile：

一个文本文件，里面包含了一系列的指令（Instruction），用于指导 Docker 如何自动构建一个镜像。

示例指令：FROM（基础镜像）, RUN（执行命令）, COPY（复制文件）, EXPOSE（暴露端口）, CMD（容器启动命令）。

Docker 的特点与优势

  • 环境一致性： “在我的机器上能跑，为什么线上不行？” —— Docker 彻底解决了这个难题。保证了从开发、测试到生产环境的绝对一致。

  • 隔离性： 容器与宿主机以及其他容器相互隔离，更安全，也不会发生依赖冲突。

  • 轻量级： 容器直接共享宿主机的操作系统内核，无需像虚拟机一样模拟整个操作系统，因此启动更快、资源占用更少。

  • 便携性和易扩展： 镜像可以非常容易地在任何地方运行，也易于实现快速的水平扩展。

Kubernetes (K8s)

什么是 Kubernetes

Kubernetes 是一个开源的容器编排引擎，用于自动化容器化应用的部署、扩缩和管理。它源自 Google，现在由云原生计算基金会（CNCF）维护。

核心概念

Kubernetes 抽象了底层基础设施，它看到的是一组计算节点（Node） 组成的集群。它的核心是声明式 API：你告诉 K8s 你期望的应用状态（如：需要运行 3 个副本），K8s 会不断地调整实际状态，直到与期望状态一致。

Pod：

K8s 管理和调度的最小单位。

一个 Pod 包含一个或多个紧密相关的容器（例如，一个应用容器和一个日志收集 sidecar 容> 器），它们共享网络和存储空间。

重要提示： 通常一个 Pod 只运行一个主容器。

Deployment：

最常用的管理无状态应用的控制器。

它定义了 Pod 的模板和副本数（Replicas）。它负责确保始终有指定数量的 Pod 副本在运行。

它实现了滚动更新和回滚，让你能无缝发布新版本。

Service：

因为 Pod 是“短暂”的（可能被重启或调度到其他节点），它的 IP 地址会变化。

Service 提供了一个稳定的网络端点（IP 地址和 DNS 名称）和负载均衡器，为一组功能相同的 Pod（通常由 Deployment 管理）提供统一的访问入口。

Ingress：

Service 通常提供的是集群内部的访问。Ingress 是管理外部访问（如 HTTP/HTTPS 流量）到集群内服务的 API 对象，它提供了路由规则（类似于 Nginx 的反向代理功能）。

ConfigMap & Secret：

用于将配置信息（如配置文件、环境变量）和敏感信息（如密码、密钥）与容器镜像解耦，从而实现应用配置的灵活管理。

Namespace：

用于在同一个物理集群中创建多个虚拟集群，实现资源和对象的逻辑隔离（例如：为开发、测试、生产环境创建不同的命名空间）。

Kubernetes 的特点与优势

  • 服务发现与负载均衡： 自动为容器组分配唯一域名和 IP，并进行负载均衡。

  • 存储编排： 可以自动挂载你选择的存储系统（本地存储、云存储等）。

  • 自动发布和回滚： 可以逐步更新应用版本，并在发现问题时立即回滚。

  • 自我修复： 重启失败的容器、替换容器、杀死不健康的容器。

  • 密钥与配置管理： 管理敏感信息和配置，无需在镜像中暴露。

  • 弹性伸缩： 根据 CPU 使用率或其他指标，自动扩缩容应用副本数量。

  • 批处理执行： 除了服务，还可以管理批处理和 CI/CD 任务。
