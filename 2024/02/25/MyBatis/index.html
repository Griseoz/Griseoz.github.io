什么是 mybatis

它是一款半自动的 ORM 持久层框架，具有较高的 SQL 灵活性，支持高级映射(一对一，一对多)，动态 SQL，延迟加载和缓存等特性，但它的数据库无关性较低

核心思想：MyBatis 的核心思想是 “SQL 与代码分离”。开发者通过 XML 或注解的方式来配置和映射 SQL 语句，将复杂的 JDBC 代码（如设置参数、处理结果集）抽象出来，使得开发者可以专注于 SQL 本身和业务逻辑，而不需要处理繁琐的底层数据库操作。

mybatis 的特点

MyBatis 之所以流行，是因为它兼具了 JDBC 的灵活性和 ORM 框架的便捷性，其主要特点如下：

  1. 轻量级且简单易学
  • 框架本身很小，API 非常简洁，学习曲线平缓。
  • 与 Hibernate 这样的全自动 ORM 框架相比，概念更少，更容易上手和理解。
  1. SQL 与代码分离，灵活性高（核心优势）

  • 手动编写 SQL：开发者需要自己编写 SQL 语句。这听起来像是个缺点，但实际上是其最大的优点。

  • 优点：你可以对 SQL 进行完全的优化和精确的控制，能编写复杂、高效的 SQL 来充分利用数据库的特性（如存储过程、高级查询、函数等）。这对于性能要求极高的项目至关重要。

  • 对比：全自动 ORM（如 Hibernate）通过 HQL 自动生成 SQL，虽然开发快，但在复杂场景下生成的 SQL 可能不够优化，调优困难。

  1. 强大的动态 SQL 功能

  • MyBatis 提供了一个强大的动态 SQL 元素，允许你在 XML 映射文件中编写灵活的、基于条件的 SQL。

  • 好处：避免了在 Java 代码中通过字符串拼接来组装 SQL，使得代码更清晰、更安全（防止 SQL 注入），也更易于维护。

  1. 提供映射标签，支持对象与数据库的 ORM 字段映射

  • 虽然需要手动写 SQL，但 MyBatis 会自动完成 查询结果集（ResultSet）到 Java 对象（POJO） 的映射。

  • 你可以通过 标签来精细地配置数据库列和对象属性之间的映射关系，支持一对一、一对多、多对多等复杂关联关系的映射。

  1. 与 Spring 等主流框架集成良好
  • MyBatis 与 Spring、Spring Boot 可以无缝集成，通过 mybatis-spring 集成包，可以非常方便地将 MyBatis 纳入 Spring 的 IOC 容器中进行管理，使用 Spring 的事务管理等特性。
  1. 支持插件扩展

  • MyBatis 提供了插件（Interceptor）机制，允许开发者编写插件来拦截和修改 MyBatis 核心的执行流程（如执行 SQL、处理参数、处理结果集等）。

  • 著名的 PageHelper 分页插件就是基于此机制实现的，可以非常方便地实现物理分页。
